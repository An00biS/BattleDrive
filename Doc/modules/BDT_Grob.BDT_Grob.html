BDT_Grob constants
A grob is an object that encapsulates an organized set of sprites which together form a pseudo-3D rotating graphical entity, useful for games. Fields
ANGLE_UPDATE_FOLLOW Designates the grob's reaction to rotation of it's parent; Grob rotates along (default)
ANGLE_UPDATE_IGNORE Ditto; Grob ignores the rotation
ANGLE_UPDATE_SKIP Ditto; Grob ignores the update, but passes it to it's child grobs.
class RootGrob extends Grob
Root object of grob tree; Can be independently positioned Fields
parentGrob Grob a pointer to the parent grob.
pegIndex number Index of the parent grob's peg this SubGrob is attached to.
displacementX The subgrob's displacement against the parent's peg (X axis)
displacementY The subgrob's displacement against the parent's peg (Y axis)
displacementH The subgrob's displacement against the parent's peg (Virtual Z axis = Y axis)
synchronizeVisualAngle boolean Modifies the visual angle to synchronize rotation with the parent grob (provided it has equal number of visual angles); If two grobs rotate unsynchronized, the parent's and child's sprites change in sequence, like "parent, child, parent, child" and the resulting "vibration" effect is very unpleasant; Only effective with parentRotated callback; Default: True if grobs have equal number of visual angles, false otherwise.
angleSyncData number Stores the angle difference between unsynchronized angle and synchronized angle; It only needs to be recalculated when this sub-grob rotates itself, otherwise it can be re-used.
angleInSync boolean Tells if the sync data are valid.
angleUpdateHandling number One of ANGLE_UPDATE constants, designating the reaction to parent's rotation.

displacementY The subgrob's displacement against the parent's peg (Y axis)
displacementH The subgrob's displacement against the parent's peg (Virtual Z axis = Y axis)
synchronizeVisualAngle boolean Modifies the visual angle to synchronize rotation with the parent grob (provided it has equal number of visual angles); If two grobs rotate unsynchronized, the parent's and child's sprites change in sequence, like "parent, child, parent, child" and the resulting "vibration" effect is very unpleasant; Only effective with parentRotated callback; Default: True if grobs have equal number of visual angles, false otherwise.
angleSyncData number Stores the angle difference between unsynchronized angle and synchronized angle; It only needs to be recalculated when this sub-grob rotates itself, otherwise it can be re-used.
angleInSync boolean Tells if the sync data are valid.
angleUpdateHandling number One of ANGLE_UPDATE constants, designating the reaction to parent's rotation.
